# UNI-T UT117C ŌĆö `ut117c`

> **State:** `untested` (ported, not bench-tested; sample vectors synthesized). **Driver:** `packages/protocol/src/drivers/ut117c.ts`. **Source:** ported from `webspiderteam/Bluetooth-DMM-For-Windows`.

## Overview

The UT117C is a UNI-T digital multimeter / clamp-capable meter exposed over BLE by the UNI-T **Smart Measure** Android app. Unlike the streaming `uni-t` family, this model is a **polled (request/response) AB-CD meter**: the meter never pushes data on its own. The app (and this driver) repeatedly writes a *"request realtime data"* command and the meter answers with **exactly one frame per request** (`ut117c.ts:7-11`).

The driver was ported from the decompiled app sources (`UT117cManager.java`, `UT117CTestDataModel.java`, `UT117CUtils.java`) ŌĆö see the upstream `webspiderteam/Bluetooth-DMM-For-Windows`. It is marked `verification: 'ported-unverified'` (`ut117c.ts:309`) because it has not been validated against physical hardware; per PLAN ┬¦6 ("Verification honesty") the function-label table is reverse-engineered, not authoritative.

Key shape:
- Transport: shared ISSC "Transparent UART" GATT service, but writes go to **WriteUUID2** (`ŌĆ”8841ŌĆ”`) (`ut117c.ts:7-8`, `311-313`).
- Framing: generic AB-CD framing ŌĆö `AB CD`, big-endian 16-bit length, payload, trailing 16-bit big-endian **additive** checksum (`ut117c.ts:13-18`).
- Decode never throws: a short/garbled/mistyped frame yields a blank `Reading` (`ut117c.ts:65-90`, `137-138`).

- Because several UNI-T models share `ISSC_SERVICE`, the registry's ISSC name-routing disambiguates by `namePrefixes` / `match()`. A first-frame `sniff` (below) is also provided to resolve a service collision on the raw byte stream.

## Handshake / session start   <polled request/response>

There is **no persistent handshake**. The meter answers one frame per poll, so the driver simply **kicks the poll loop once** and relies on the session keep-alive + `onRequest` to re-arm:

```ts
// handshake ŌĆö kick the poll once (ut117c.ts:321-323)
async handshake(io) {
  await io.write(CMD.POLL);   // AB CD 00 04 05 00 01 81
}

// onRequest ŌĆö re-send POLL on every answer (measurement frame) or control ACK (ut117c.ts:328-331)
onRequest(frame, io) {
  void frame;
  void io.write(CMD.POLL);
}
```

This mirrors the app's `startRealTimeRead` loop and the *"ACK resumes polling"* behaviour in `UT117cManager` (`ut117c.ts:326-327`). Frames classified as either `measurement` (type 2) or `control` (type 1 ACK) drive the next poll, keeping the request/response cycle alive.

### Command frame construction

Each control/poll frame is 8 bytes, built byte-for-byte by `control(cmd, arg)` (`ut117c.ts:42-47`):

```
AB CD 00 04 <cmd> <arg> 01 <chk>
[0] [1] [2][3] [4]   [5]   [6]  [7]
```

- `[0..1]` = `AB CD` sync header (`SOF_H`/`SOF_L`, `ut117c.ts:29-30`).
- `[2..3]` = `00 04` big-endian length.
- `[4]` = `cmd`, `[5]` = `arg`.
- `[6]` = `0x01` trailing marker (a literal constant, **excluded** from the checksum).
- `[7]` = checksum = **low byte** of the 16-bit additive sum of the six bytes `b[0..5]` (header + length + cmd + arg) ŌĆö `(SOF_H + SOF_L + 0x00 + 0x04 + cmd + arg) & 0xff` (`ut117c.ts:38-46`).

The checksum *deliberately excludes the trailing `0x01` marker*. Verified in source against the captured poll `0x81` (sum `b[0..5] = 0x181`) and hold `0xE8` frames; all nine commands round-trip (`ut117c.ts:40-41`).

## Frame format   <onDataReceive parse>

Generic AB-CD framing, big-endian length + 16-bit additive checksum (`ut117c.ts:13-24`):

```
[0]   = 0xAB                    sync header high (SOF_H)
[1]   = 0xCD                    sync header low  (SOF_L)
[2..3]= LEN (big-endian)        payload byte count starting at the type byte b[4]
[4]   = packet type             1 = command ACK, 2 = realtime measurement data (TYPE_DATA)
ŌĆ”payloadŌĆ”
last 2 bytes = checksum = ╬Ż(all preceding bytes) & 0xFFFF, big-endian
```

For a **TYPE-2 measurement frame**, `LEN = 0x12` (18) and the total frame is **24 bytes** (`DATA_LEN = 0x12`, `DATA_TOTAL = 4 + 18 + 2 = 24`, `ut117c.ts:35-36`):

| Offset | Field | Notes (`ut117c.ts:19-24`) |
| --- | --- | --- |
| `[0..1]` | `AB CD` header | |
| `[2..3]` | `00 12` length | big-endian = 18 |
| `[4]` | type = `2` | `TYPE_DATA` (`ut117c.ts:32`) |
| `[5]` | `funcID` | function code ŌåÆ label map |
| `[6]` | `rangIndex` | ASCII digit (`value ŌłÆ 0x30`); also NCV strength selector |
| `[7]` | `hzRangIndex` | Hz range index (parsed in app; not surfaced by driver) |
| `[8..14]` | 7-byte ASCII value string | space-padded, may have leading `-` |
| `[15..17]` | 3-byte ASCII unit string | `'o'` ŌåÆ `'╬®'` remap |
| `[18]` | `olFlag` | `0x31`='OL', `0x32`='-OL' |
| `[19]` | `maxminFlag` | max/min record mode (read by app; not surfaced as a flag here, `ut117c.ts:207`) |
| `[20]` | `flag1` | hold / rel / lowBattery bits |
| `[21]` | `flag2` | AC/DC / auto / hvWarning bits |
| `[22..23]` | checksum | 16-bit big-endian additive over `b[0..21]` |

### Framer / sync / resync (`Ut117cFramer`, `ut117c.ts:231-275`)

- Accumulate bytes, **sync** on `AB CD` (discard non-`AB` bytes; if `b[1] != CD`, drop and re-sync, `ut117c.ts:262-274`).
- Need Ōēź 5 bytes (`AB CD LEN_H LEN_L type`) before reading length (`ut117c.ts:239`).
- `len = (b[2] << 8) | b[3]`; `total = 4 + len + 2` (`ut117c.ts:240-241`).
- Bogus length guard: if `len < 1` or `total > 64`, shift one byte and re-sync (false header in noise, `ut117c.ts:242-245`).
- Wait if `buf.length < total` (frame split across notifications, `ut117c.ts:246`).
- **Checksum gate on measurement frames only:** if `frame[4] === TYPE_DATA` and `!checksumOk(frame)`, shift one byte and re-sync rather than emit a corrupt reading (`ut117c.ts:248-251`).
- Tolerates split *and* coalesced notifications; loops to emit multiple frames per `push` (`ut117c.ts:237-256`).

### Checksum (`checksumOk`, `ut117c.ts:278-286`)

16-bit big-endian additive checksum over **every byte preceding the trailing 2 checksum bytes**:

```
sum = (╬Ż frame[0 .. len-3]) & 0xFFFF
ok  = (sum >> 8) == frame[len-2]  &&  (sum & 0xFF) == frame[len-1]
```

Returns `false` for frames shorter than 3 bytes.

### Classification (`classify`, `ut117c.ts:222-224`)

```
frame[4] === TYPE_DATA (2) ŌåÆ 'measurement'
otherwise (incl. type 1 ACK) ŌåÆ 'control'
```

Both kinds trigger an `onRequest` re-poll.

### Value & unit strings

- `valueStr = asciiString(bytes, 8, 7, remapOhm=false)` ŌĆö 7 ASCII bytes, trimmed (`ut117c.ts:157`).
- `displayUnit = asciiString(bytes, 15, 3, remapOhm=true)` ŌĆö 3 ASCII bytes; `'o'` (`0x6f`) ŌåÆ `'╬®'` (`ut117c.ts:158`, `121-129`).

Numeric test: `NUMERIC = /^-?\d*\.?\d+$/` (`ut117c.ts:131`); `numeric = valueStr.length > 0 && NUMERIC.test(valueStr)` (`ut117c.ts:182`).

### Overload (`ut117c.ts:183-190`)

Overload applies **only to numeric readings** ŌĆö `olFlag` overrides the value string:

```
overload    = numeric && (olFlag === 0x31 || olFlag === 0x32)
displayText = overload ? (olFlag === 0x32 ? '-OL' : 'OL') : valueStr
displayValue = (numeric && !overload) ? Number(valueStr) : null
```

### Base-value normalization (`ut117c.ts:192-193`)

```
{ base: baseUnit, exp } = unitInfo(displayUnit)
baseValue = displayValue === null ? null : displayValue * 10 ** exp
```

`unitInfo` (`types.ts:168-174`) splits a metric prefix (`n/┬Ą/m/k/M` ŌåÆ `ŌłÆ9/ŌłÆ6/ŌłÆ3/3/6`) off the unit only when a base char remains; e.g. `k╬®` ŌåÆ base `╬®`, exp `3`; bare `V`/`╬®`/`%` stay put.

### AC/DC (`ut117c.ts:150-155`)


Flags are still decoded (hold/rel/auto/lowBattery/hvWarning) using the same bits as above.

## Sniffer

`looksLikeUt117cFrame(bytes)` resolves the shared-ISSC-service collision on the raw first frame (`ut117c.ts:290-300`, wired as `sniff` at `ut117c.ts:346`). Returns `true` iff all hold:

```
bytes.length === 24 (DATA_TOTAL)
bytes[0] === 0xAB
bytes[1] === 0xCD
bytes[2] === 0x00
bytes[3] === 0x12 (DATA_LEN)
bytes[4] === 2    (TYPE_DATA)
checksumOk(bytes)
```
